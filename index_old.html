🚀 GitHub Models Actions Examples 🤖

Intelligent code automation using GitHub Models within GitHub Actions workflows

✨ AI-Powered Workflows

This repository demonstrates practical examples of integrating GitHub Models into GitHub Actions workflows for intelligent code automation. Each workflow showcases different AI capabilities for enhancing your development process.

🧠 AI Code Complexity Analyzer

Automatically analyzes JavaScript files for code complexity using GitHub Models and generates detailed reports.

  • Complexity scoring and analysis
  • AI-powered recommendations
  • Automatic report generation
  • Timestamped analysis history

📝 AI PR Description Generator

Generates informative pull request descriptions from code diffs using GitHub Models AI analysis.

  • Automatic PR enhancement
  • Code diff analysis
  • Impact assessment
  • Smart summarization

📊 AI Project Summary Generator

Creates comprehensive project documentation for AI coding agents by analyzing codebase structure and components.

  • Codebase understanding
  • Architecture documentation
  • AI agent assistance
  • Automatic updates

⚡ Quick Setup

Get started with GitHub Models in your Actions workflows in just a few steps:

Fork this repository

Click the "Fork" button to create your own copy of this repository with all the example workflows.

Fork Repository

Generate GitHub Personal Access Token

Create a PAT with appropriate permissions for GitHub Models API access.

• Go to GitHub Settings → Developer settings → Personal access tokens • Generate new token with 'read:org' and 'user' scopes • Copy the token for the next step

Configure Repository Secrets

Add the required secrets to your repository for GitHub Models integration.

Repository Settings → Secrets and variables → Actions Required secrets: • OPENAI_API_KEY: Your GitHub PAT • OPENAI_API_ENDPOINT: https://models.inference.ai.azure.com • OPENAI_MODEL: gpt-4o (or your preferred model)

Test the workflows

Push changes to trigger the workflows or run them manually to see GitHub Models in action!

# The workflows will automatically: • Analyze code complexity on pushes • Generate PR descriptions when PRs are opened • Create project summaries on main branch updates
